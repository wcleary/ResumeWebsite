About the LuaGuard Project

Lua is a small scripting language, born in 1993. It is designed for embedded applications, aimed to provide flexible expansion and customization features for the application. Meanwhile the compilation speed of Lua is much faster than other scripting languages. Currently a lot of games, such as World of Warcraft, Angry Birds, VOCALOID3, are using Lua as its scripting language.

Reverse engineering can be a part of the product design process. Existing products can be used to derive a product design such as the data model. Through reverse engineering, it is possible to deduce the original technology or source code from the existing products, which poses a threat to protecting intellectual property.

This project processes Lua source code with methods such as function renaming, return value, and junk code obfuscation. After processing, it is inherently more difficult for a developer to read the source code, and therefore it is harder to reverse engineer. In this way, we can protect intellectual property more effectively.

The system is designed as a source-to-source translator with three main components: a front-end, obfuscators (transformations), and a parser and unparser. The front-end allows for users to provide Lua source files as input in addition to optional parameters that modify the behavior of the obfuscators and output. The front-end includes a common set of operations, control and config, that all user interfaces, e.g. command line or GUI, utilize for the code transformation process. The parser takes as input a Lua source file and produces an abstract syntax tree (AST). The obfuscations are part of a plug-in framework that allows for new obfuscations to be added. All obfuscators take as input the AST produced by the parser and return a modified AST. The AST may go through any number of transformations after which it is passed to the unparser. The unparser produces the Lua program that is represented by the AST given as input.
